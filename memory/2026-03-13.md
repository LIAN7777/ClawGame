### 🐛 紧急修复：游戏启动画面异常

**用户反馈：**
游戏完全不可用，界面是大片色块堆叠，没有人物、NPC，没有任何可操作空间，连退出键都没反应。

**问题分析：**
1. **模块导入路径错误**：`main.py` 使用 `from config import config`，但运行 `python -m src.main` 时 Python 路径不正确
2. **相机尺寸错误**：`Game` 类使用窗口尺寸（800x600）初始化相机，但实际渲染到内部表面（400x300）
3. **相机初始位置问题**：相机初始位置是 (0,0)，玩家出生在 (240,336)，且使用平滑跟随，第一帧玩家不在视口内

**修复内容：**
- `src/main.py`：添加 Python 路径设置，支持 `python -m src.main` 运行
- `src/game/game.py`：
  - 使用 `config.internal_width/height` 替代 `screen.get_width/height`
  - 相机跟随时立即调用 `update()` 定位
  - 添加 `from config import config` 导入
- `tests/test_game.py`：更新测试以匹配新的 Game 类行为

**验证结果：**
- 66 个测试全部通过
- 相机位置：(52.0, 84.0) - 合理
- 相机尺寸：400x300 - 正确匹配内部渲染尺寸
- 玩家在视口内：True
- 所有 3 个 NPC 在视口内：True

### 🐛 修复移动键无响应和FPS显示问题

**用户反馈：**
可以看到场景和人物了，四个不同颜色但风格完全相同的角色。按移动键没有反应，FPS显示一直为0。

**问题分析：**
1. **玩家出生点碰撞问题**：出生点在门前一格(7,10)，碰撞盒覆盖了墙壁格子(8,11)，导致向下和向右移动被阻止
2. **FPS计算错误**：`Game` 类有独立的 `_clock` 但从未调用 `tick()`，`get_time()` 始终返回0
3. **主角辨识度低**：主角和NPC使用相似的粉色调，难以区分

**修复内容：**
1. `src/game/scene.py`：
   - 出生点从门前一格改为门前两格，避免与墙壁碰撞
   
2. `src/game/game.py`：
   - 移除 `_clock`，改用 `_last_dt` 存储外部传入的 delta time
   - `update()` 方法接收 `dt` 参数
   - `_render_debug_info()` 使用 `1/dt` 计算FPS
   - 调试文字改为白色

3. `src/main.py`：
   - 在游戏循环中计算 `dt` 并传给 `game.update()`

4. `src/game/entity/player.py`：
   - 主角配色改为更鲜艳的珊瑚粉
   - 添加金色小皇冠作为主角标识

**验证结果：**
- 玩家位置：(240, 304) - 不与墙壁碰撞
- 四个方向移动测试：全部返回 True
- FPS计算：dt=0.016 → FPS=62 ✅
- 66个测试全部通过

2. `src/game/interaction.py`：

   - 添加 `update_cooldown()` 方法
   - `try_interact()` 检查冷却时间

3. `src/game/entity/npc.py`：
   - 添加 `DIALOGS` 字典，不同颜色NPC有不同对话
   - `interact()` 方法防止重复触发
   - 气泡显示时间从2秒延长到3秒

4. `src/game/game.py`：
   - 移除 `_render_debug_info()` 调用
   - 调用 `interaction_system.update_cooldown(dt)`

**NPC对话内容：**
- 绿色: "你好呀~", "今天天气真好！", "需要帮忙吗？"
- 蓝色: "嘿~", "有什么新鲜事吗？", "一起来玩吧！"

### ✨ 修复中文字体显示问题，添加主角加速/减速效果

**用户反馈：**
1. NPC字幕显示问题，字全部显示为矩形框，像是兼容性问题
2. 主角奔跑和停下应该更加符合物理规律，有加速的过程

**问题分析：**
1. **中文字体问题**：`pygame.font.Font(None, 20)` 使用默认字体不支持中文字符
2. **移动缺乏物理感**：按键直接变速，没有加速/减速过程

**修复内容：**
1. `src/game/entity/npc.py`：
   - 添加 `_get_chinese_font()` 函数自动检测系统中文字体
   - 使用 NotoSansCJK-Bold.ttc 字体渲染中文
   - 气泡字体大小调整为16px

2. `src/game/entity/player.py`：
   - 添加 `ACCELERATION = 400` 和 `DECELERATION = 600` 常量
   - 添加 `actual_vx` 和 `actual_vy` 表示实际速度
   - `vx` 和 `vy` 改为目标速度（由用户输入设置）
   - 添加 `_update_velocity()` 方法平滑过渡实际速度
   - `_update_anim_state()` 使用实际速度判断移动状态

**物理效果：**
- 加速：从静止到最大速度需要约 0.25 秒（行走）或 0.45 秒（奔跑）
- 减速：从最大速度到停止需要约 0.17 秒（行走）或 0.3 秒（奔跑）
- 停止按键时自然减速停止，不再瞬间停下

**验证结果：**
- 中文字体可以正常渲染 ✅
- 66个测试全部通过 ✅
- 主角移动有加速/减速效果 ✅

### 🐛 修复中文字体显示问题（第二次）

**用户反馈：**
字体显示仍有问题，还是全是长方形框。

**问题分析：**
1. 字体路径列表中有错误路径（`/usr/share/fonts/truetype/noto/` 不存在）
2. 字体加载优先级不合理，.ttc 文件可能比 .ttf 文件更难处理

**修复内容：**
1. `src/game/entity/npc.py`：
   - 优先使用 `DroidSansFallbackFull.ttf`（简单可靠的 .ttf 文件）
   - 修复错误的字体路径
   - 添加异常处理，避免静默失败

2. `src/game/interaction.py`：
   - 交互提示 "Press E to talk" 也使用中文字体
   - 添加 `_get_font()` 函数

**字体优先级：**
1. `/usr/share/fonts/truetype/droid/DroidSansFallbackFull.ttf` ✅
2. `/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc`
3. `/usr/share/fonts/opentype/noto/NotoSansCJK-Bold.ttc`

**验证结果：**
- 中文渲染测试：尺寸 (64, 21) ✅
- 66个测试全部通过 ✅
